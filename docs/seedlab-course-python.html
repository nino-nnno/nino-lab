Python 探案学院

告别 print("hello world")。用 Python 写天气机器人、文件整理脚本、爬虫小工具。让代码真正干点活。

从 Scratch 到 Python：为什么这个跨越最容易翻车

很多孩子从 Scratch 切换到 Python 的时候会遭遇巨大的挫败感。原因很简单：Scratch 是所见即所得——拖一个积木，小人马上动一下。Python 是纯文本——写十行代码，屏幕上可能只出现几个字，甚至什么也没有。这种"延迟满足"是编程学习的第一个真正门槛。

市面上的 Python 课，大多沿用成人教程的逻辑：先讲数据类型，再讲 if/else，再讲 for 循环……每节课一个知识点，期末做一个大作业。问题是——孩子根本不知道这些东西能用来干什么。学了一堆语法，不知道能解决什么问题。

我们的做法完全反过来。

"探案"模式：先有谜题，再找工具

每节课不是一个"知识点"，而是一个"案件"：

  • 案件一：自动整理下载文件夹。下载文件夹里有几百个文件混在一起。怎么让 Python 自动把图片归到一个文件夹、文档归到另一个？——引出文件操作、字符串处理和 if 判断。
  • 案件二：写一个天气查询机器人。输入城市名，返回今天的温度和天气描述。——引出 API 调用、JSON 解析、函数定义。
  • 案件三：监控某东商品价格。每隔半小时自动抓取一个商品的价格，降价了给自己发邮件。——引出爬虫基础、定时任务、邮件发送。
  • 案件四：做一个课堂抽答器。随机点一个同学的名字并显示在屏幕上，点过的不会重复。——引出列表、随机模块、Tkinter 图形界面。

每个案件都是真实的、能用的问题。孩子不是"学 Python"，而是"用 Python 破案"。语法是顺带学的。

果果的"作业提醒机器人"

12 岁的果果经常忘写作业。她在 Python 课上写了一个脚本：每天晚上 8 点自动弹窗提醒"该写作业了！"，弹窗上有两个按钮——"好的"和"再睡五分钟"。点"再睡五分钟"，五分钟后继续弹。这个项目用到了 datetime 模块、Tkinter 弹窗、while 循环和条件判断。从有想法到跑通，两节课。现在她全家都在用这个脚本做各种提醒。

课程进度

32 节课四个单元：

  • 单元一（1-8 课）· 破案工具包：变量、输入输出、条件判断、循环。用五个小案件练手（猜数字、BMI 计算器、密码强度检测、自动出题器、文本分析）。
  • 单元二（9-16 课）· 进阶侦查：列表、字典、函数、文件读写。案件升级为批量改名、成绩统计分析、日记加密、网页数据抓取。
  • 单元三（17-24 课）· 联网办案：API 调用、JSON 解析、邮件发送、定时任务。天气查询、新闻推送、价格监控——写出来真的能用的东西。
  • 单元四（25-32 课）· 毕业大案：每个孩子自选一个真实问题，用四周时间独立开发完成。老师只做代码审查。常见选题：个人网站、自动化脚本、数据可视化、AI 聊天机器人。

长远来看

Python 目前是中国最主流的编程入门语言，也是信奥赛（NOI）之外最实用的选择。学完这门课，孩子具备的能力包括：独立编写自动化脚本、理解 API 和数据交互、使用第三方库解决问题、阅读英文技术文档的基本能力。这些不是"编程课学的"，是"干活的工具"。
